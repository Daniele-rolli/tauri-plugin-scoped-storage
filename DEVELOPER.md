# Developer Guide

This document is for contributors and maintainers working on `tauri-plugin-scoped-storage`. For end-user documentation see [README.md](./README.md).

---

## Goals

- Give Tauri 2 apps safe, user-approved folder access on Android and iOS.
- Keep the Rust command layer as the single canonical validation boundary.
- Return stable, structured error payloads to guest JavaScript regardless of which platform generated the failure.
- Favour conservative behaviour over permissive behaviour for filesystem mutations.

## Architecture

The plugin has three layers that communicate in one direction: Rust validates → native executes → result surfaces to JS.

### 1. Rust Command Boundary (`src/commands.rs`)

Every command that reaches `commands.rs` has already been deserialized from the IPC payload by Tauri. Before any native call, `commands.rs`:

- Strips and validates `folder_id` (must be non-empty).
- Normalises relative `path` values via `normalize_relative_path` in `src/path.rs`.
- Rejects identical source/destination pairs for `copy`, `move`, and `rename`.
- Returns `ScopedStorageError` directly on validation failure — no native round-trip.

The native mobile layer performs equivalent checks defensively, but Rust owns the primary policy. If validation rules change, update Rust first.

### 2. Native Platform Implementation

**Android** (`ScopedStoragePlugin.kt`):
- All file I/O runs on `ioScope` (a per-instance `Dispatchers.IO + SupervisorJob()`).
- Folder handles are persisted as `content://` tree URIs via `takePersistableUriPermission`.
- File navigation uses `DocumentFile.fromTreeUri` + `findFile` walking; there is no direct path access.
- Errors thrown during native operations are caught and normalised through `Invoke.rejectScoped`, which emits a `SCOPED_STORAGE_ERROR:<CODE>:<message>` string that Rust's `ScopedStorageError::from_native_message` can parse back into a typed variant.

**iOS** (`ScopedStoragePlugin.swift`):
- Folder handles are stored as security-scoped bookmarks (`bookmarkData`), which survive across app restarts.
- Every command calls `url.startAccessingSecurityScopedResource()` for the duration of its operation and releases it in a `defer` block.
- Stale bookmarks are detected and refreshed transparently inside `withFolderURL`.
- All writes and copies use `NSFileCoordinator` for safe concurrent access and iCloud Drive compatibility.
- Errors follow the same `SCOPED_STORAGE_ERROR:<CODE>:<message>` convention via `scopedStorageRejectMessage`.

### 3. Guest JavaScript Bindings (`guest-js/index.ts`)

- Provides lightweight argument validation (non-empty strings, `Uint8Array` type checks) for early developer feedback.
- Wraps every `invoke` call in `invokeCommand`, which normalises thrown values into `ScopedStorageError` instances.
- `dist-js/` is the build output published to npm — it is generated from `guest-js/` and should never be edited by hand.

## Error Contract

### Structured Error Format

Native code should emit errors in this exact format:

```
SCOPED_STORAGE_ERROR:<CODE>:<human-readable message>
```

For example:
```
SCOPED_STORAGE_ERROR:NOT_FOUND:Path not found: notes/hello.txt
```

`ScopedStorageError::from_native_message` in `src/error.rs` parses this format. If the prefix is absent it falls back to heuristic keyword matching (e.g. "cancel", "permission", "not found") before defaulting to `NATIVE_ERROR`.

### Stable Error Codes

Defined in `src/error.rs` and mirrored in `ScopedStorageSupport.kt` / `ScopedStorageSupport.swift`:

| Code | Rust variant |
|------|-------------|
| `UNSUPPORTED` | `ScopedStorageError::Unsupported` |
| `INVALID_PATH` | `ScopedStorageError::InvalidPath(String)` |
| `FOLDER_NOT_FOUND` | `ScopedStorageError::FolderNotFound(String)` |
| `NOT_FOUND` | `ScopedStorageError::NotFound(String)` |
| `ALREADY_EXISTS` | `ScopedStorageError::AlreadyExists(String)` |
| `PERMISSION_DENIED` | `ScopedStorageError::PermissionDenied` |
| `CANCELLED` | `ScopedStorageError::Cancelled` |
| `INVALID_ARGUMENT` | `ScopedStorageError::InvalidArgument(String)` |
| `IO_ERROR` | `ScopedStorageError::Io(String)` |
| `NATIVE_ERROR` | `ScopedStorageError::Native(String)` |

Do not invent new codes without updating all three layers (Rust, Kotlin, Swift) and the README error table.

### JS Serialisation

`ScopedStorageError` implements `serde::Serialize` by delegating to `ErrorPayload { code, message }`. This is what JS receives as the thrown value:

```json
{ "code": "NOT_FOUND", "message": "not found: notes/hello.txt" }
```

`invokeCommand` in `guest-js/index.ts` wraps this into a typed `ScopedStorageError` instance so callers can use `isScopedStorageError` and `e.code`.

## Path Policy

`normalize_relative_path` in `src/path.rs` is the enforcer. It:

1. Trims whitespace.
2. Replaces `\` with `/`.
3. Collapses empty segments and `.` components.
4. Rejects `..` (returns `InvalidPath`).
5. Rejects paths starting with `/` or `~`.
6. Rejects paths whose first segment contains `:` (URI-style and drive-style prefixes).
7. Returns an empty string for a whitespace-only input (callers that require a non-empty path use `normalize_required_path`).

The Kotlin `ScopedStoragePaths.split` and Swift `ScopedStoragePath.split` methods apply the same rules independently.

**Do not relax these rules without updating all three layers and the README path rules section.**

## Adding or Changing a Command

Follow these steps in order to keep all layers consistent:

1. **Rust model** — add or update request/response structs in `src/models.rs`.
2. **Rust command** — add the handler in `src/commands.rs` with full validation.
3. **`src/lib.rs`** — register the new command in `tauri::generate_handler![]`.
4. **`build.rs`** — add the command name string to the `COMMANDS` array.
5. **Desktop stub** — add a method to `src/desktop.rs` that returns `Err(ScopedStorageError::Unsupported)`.
6. **Mobile bridge** — add the method to `src/mobile.rs` that calls `run_mobile` with the right command name.
7. **Android** — implement the `@Command` method in `ScopedStoragePlugin.kt`. Add a corresponding `@InvokeArg` class if needed.
8. **iOS** — implement the `@objc public func` in `ScopedStoragePlugin.swift`. Add a `Decodable` args struct if needed.
9. **Guest JS** — add the exported function to `guest-js/index.ts`. Rebuild with `pnpm build`.
10. **Permissions** — run `cargo tauri permission add` or manually create a `.toml` in `permissions/autogenerated/commands/`. Add the new permission to `permissions/default.toml` if it should be in the default set.
11. **Docs** — update `README.md` (API reference table and example) and `docs/manual-test-notes.md`.

**Never ship a command that is implemented on only one mobile platform without explicitly documenting the asymmetry.**

## Safety Rules

These invariants are intentional. Do not change them without updating Rust validation, tests, all native layers, and the README simultaneously.

| Rule | Enforced in |
|------|-------------|
| Paths must be relative | `src/path.rs`, `ScopedStoragePaths`, `ScopedStoragePath` |
| `..` is rejected | All three layers |
| Absolute and URI-style prefixes are rejected | All three layers |
| Blank folder IDs are rejected | `src/commands.rs` (`normalize_folder_id`) |
| Required paths must not be empty after normalisation | `src/commands.rs` (`normalize_required_path`) |
| `copy`, `move`, `rename` reject identical source and destination | `src/commands.rs` (`ensure_distinct_transfer`) |
| `truncate len` must be non-negative | `src/commands.rs` (type is `u64`); Android validates before coroutine dispatch |

## Android-Specific Notes

### Storage Access Framework Limitations

- There is no direct POSIX path access. All navigation is done by walking `DocumentFile.findFile` segments.
- SAF does not support atomic rename across different parent directories. Cross-parent moves use copy-then-delete via `moveEntry`.
- Same-parent renames use `DocumentFile.renameTo`, which is atomic on most Android filesystem providers.
- `DocumentFile.length()` returns 0 for directories — always check `isFile` before reading `size`.

### Coroutine Scope

`ioScope` is an instance variable of `ScopedStoragePlugin` tied to a `SupervisorJob`. It is cancelled in `onDestroy()` to prevent coroutine leaks if the plugin is torn down before an in-progress operation completes. Individual child coroutine failures do not cancel the scope.

### MIME Types

Android requires a MIME type when creating files via `DocumentFile.createFile`. If the caller does not supply one, `ScopedStorageMimeTypes.forPath` infers it from the file extension. Unknown extensions fall back to `application/octet-stream`.

## iOS-Specific Notes

### Security-Scoped Resources

Every call to `withFolderURL` bookends the operation with `startAccessingSecurityScopedResource` / `stopAccessingSecurityScopedResource`. Access is stopped in a `defer` block even if the operation throws. Do not call these methods manually outside of `withFolderURL`.

### Bookmark Staleness

iOS may mark a bookmark as stale if the underlying file system item was moved by the OS (e.g. iCloud reorganisation). `withFolderURL` detects this via the `bookmarkDataIsStale` out-parameter and silently refreshes the stored bookmark using `IOSFolderStore.update`.

### File Coordination

All write, append, and truncate operations go through `NSFileCoordinator.coordinate(writingItemAt:options:)`. Copy operations use the two-URL coordinator form. This is required for correct behaviour on iCloud Drive and when other processes might concurrently access the same files.

### Move vs. Copy-then-Delete

`coordinatedMove` attempts `FileManager.moveItem` inside a coordinated write. If the move fails (e.g. cross-volume), the error is rethrown — the caller (`move()`) catches it and retries with `copyThenDelete`. This avoids the `ALREADY_EXISTS` crash that would occur if copy-then-delete were attempted after a partially-completed coordinator move.

## Manual Testing

Use the example app (`examples/tauri-app`) on a physical device. Emulators may not replicate SAF permission behaviour accurately on Android.

At minimum, cover:

- Folder pick → restart app → `listFolders` returns the same handle.
- Re-select a saved handle → `readDir`, `readTextFile`, `writeTextFile` all work.
- `writeTextFile` with `recursive: true` creates intermediate directories.
- `appendTextFile` appends without overwriting.
- `writeFile` / `readFile` round-trips exact binary byte sequences.
- `mkdir` with `recursive: true` creates nested directories.
- `copy` creates an independent copy; modifying the copy does not affect the source.
- `move` removes the source after the destination is created.
- `rename` within the same parent directory.
- `truncate` shrinks a file and truncate extends a file with zero bytes.
- `removeFile` and `removeDir` (recursive) succeed and the entries are gone from `readDir`.
- `exists` returns `false` for a deleted path.
- Cancelling the folder picker returns `CANCELLED`.
- Passing an invalid path (e.g. `../escape`) returns `INVALID_PATH`.
- Passing a missing folder ID returns `FOLDER_NOT_FOUND`.
- Reading a non-existent file returns `NOT_FOUND`.
- Copying to an existing destination returns `ALREADY_EXISTS`.

Detailed notes per platform live in [docs/manual-test-notes.md](./docs/manual-test-notes.md).
